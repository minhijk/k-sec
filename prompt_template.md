# 🧠 출력 지침 (사용자에게 표시되는 부분)

## 1. 주요 발견 사항
- 위험도별(🔴 Critical, 🟠 High, 🟡 Medium)로 요약
- 각 항목 앞에 [CIS], [NIST] 등 출처 유형 명시  
- 각 문장 끝에 근거 번호 [n] 명시

## 2. 현재 설정 문제
- ⚠️ 실제 취약한 설정만 나열 (안전한 설정은 제외)  
- YAML 경로 정확히 표시  
- “→ [위험 설명] [근거번호]” 형태로 작성  

## 3. 권장 수정안
- 수정 이유를 1~2문장으로 명시  
- **수정 전** / **수정 후** YAML 코드 블록을 각각 제시  
  - 백틱 3개 + yaml  
  - 들여쓰기 2칸 (탭 금지)
  - 코드 블록 전후 빈 줄 한 줄 유지  

## 4. 추가 권장사항
- 최소 권한 원칙, seccompProfile, capability 설정 등 구체적 조치  
- 이모지 및 위험도 레벨 표시 유지
